# cite_yagodka.org

Репозиторий статического сайта `yagodka.org`.

## Структура

- `index.html` — публичная входная страница сайта: описание проекта, возможности, скачивание клиентов и отдельная кнопка входа в web-клиент.
- `assets/` — общие ассеты (картинки/иконки/стили).
- `pages/` — статические страницы (если есть).
- `web/` — собранный web/PWA клиент Yagodka, если репозиторий публикуется как цельный статический сайт.

## Публичная схема

- Корень `https://yagodka.org/` — сайт проекта.
- `https://yagodka.org/web/` — отдельный вход в web-клиент.
- `https://yagodka.org/downloads/` — сгенерированные файлы клиентов и checksums.

## Как обновлять web/PWA

Сборка web‑клиента и downloads готовятся из основного проекта `yagodka/project_01`:

1) Собрать web‑клиент в `project_01/client-web`.
2) Выполнить `make web-www-build` из `project_01`.
3) Проверить, что `www/index.html` — сайт проекта, а `www/web/index.html` — web-клиент.
4) При необходимости обновить этот репозиторий и/или выполнить отдельный deploy.

`make web-www-build` копирует корневой сайт из `cite_yagodka.org`, web‑клиент в `www/web/`, а текущие клиенты в `www/downloads/`.

## Безопасность

- Не добавляйте в репозиторий ключи, пароли, `service.env` и любые приватные конфиги.
- Не храните персональные данные пользователей/ID в статических файлах.
- Debug APK и unsigned desktop ZIP можно использовать для проверки, но публичная публикация должна идти через отдельную подписанную сборку.
